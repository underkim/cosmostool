OpenC3 Developer Toolkit — 사용 설명서

버전 0.1.0  |  OpenC3 COSMOS 6.10.4 지원

이 툴킷은 WSL 또는 SSH로 연결된 Linux 환경에서 OpenC3 COSMOS를 개발·운영하는 분들을 위한 Windows 데스크톱 애플리케이션입니다.

1. 빠른 시작

  1. Settings 탭(⚙️)으로 이동해 연결 프로파일을 추가합니다.
  2. 프로파일을 선택하고 Connect를 클릭합니다.
  3. 연결에 성공하면 상태 바가 🟢 Connected로 바뀝니다.
  4. 각 탭에서 Docker, Doctor, Plugin 등 기능을 사용합니다.
💡 연결 전에도 앱은 실행됩니다. 연결되지 않은 상태에서 명령을 실행하면 "Not connected" 오류가 반환됩니다.

2. 연결 설정 (Settings ⚙️)

Quick WSL (권장)

Windows에 설치된 WSL 배포판을 자동으로 감지해 프로파일을 바로 만듭니다.

  1. Create WSL Profile 버튼 클릭
  2. 배포판이 자동으로 감지되면 곧바로 프로파일이 생성됩니다.
  3. 감지에 실패하면 Use <배포판> Defaults(기본값으로 계속) / Create Custom Profile(직접 입력) / Cancel 중 선택하는 대화상자가 표시됩니다.

연결 프로파일 직접 추가

  1. Create Custom Profile 버튼 클릭 → 오른쪽 폼에 정보 입력
  2. Mode 선택: WSL 또는 SSH
  3. Save Profile 클릭

WSL 연결

항목설명예시

먼저 Windows 터미널에서 ssh user@host를 실행해 지문을 수락하세요.

3. Dashboard (📊)

연결된 환경의 전반적인 상태를 한눈에 확인합니다.

  • Connection — 현재 연결 상태
  • Docker — Docker 데몬 실행 여부 및 실행 중인 컨테이너 수
  • OpenC3 Version — 원격의 OpenC3 버전
  • CPU / Memory / Disk — 실시간 리소스 사용률 (자동 갱신)

상단 메뉴 View → Refresh 또는 F5로 수동 새로고침.

4. Docker Manager (🐳)

원격 환경의 Docker 컨테이너를 관리합니다.

컨테이너 목록

  • ↻ Refresh — 목록 새로고침
  • 열: Name, Image, Status, Ports
  • 행을 선택하면 하단 버튼이 활성화됩니다.

컨테이너 작업

버튼동작
▶ Start선택한 컨테이너 시작
■ Stop선택한 컨테이너 중지
↺ Restart선택한 컨테이너 재시작
✕ Remove선택한 컨테이너 삭제 (확인 대화상자 표시)
📋 Logs최근 로그 200줄을 하단 패널에 표시

5. System Doctor (🩺)

OpenC3 환경의 건강 상태를 11개 항목으로 점검합니다.

사용법

  1. 🩺 Run Health Checks 클릭
  2. 각 항목이 순차적으로 실행되며 상단 라벨에 진행 상황이 표시됩니다.
  3. 완료 후 결과 테이블에 Pass / Warning / Fail이 표시됩니다.
  4. 테이블 행을 선택하면 하단에 조치 제안이 표시됩니다.

점검 항목

  • Docker 설치 여부
  • Docker 데몬 실행 상태
  • OpenC3 compose 파일 존재
  • CPU 사용률 (경고: 80% 이상, 실패: 95% 이상)
  • 메모리 여유 공간
  • 디스크 여유 공간
  • 네트워크 연결성
  • OpenC3 API 응답
  • 필수 포트(2900, 2943) 오픈 여부
  • 시스템 시간 동기화
  • 로그 파일 크기

6. Workspace (🧩)

플러그인 관리와 CMD/TLM 파일 편집을 한 화면에서 처리합니다.

플러그인 관리

버튼동작
↻ Refresh설치된 플러그인 목록 새로고침
Install Gem.gem 파일 선택 후 원격에 설치
More ▾ → Remove선택한 플러그인 제거 (확인 대화상자)
Check로컬 플러그인 파일/디렉터리 구조 검증
Build Gem선택한 플러그인을 gem 파일로 빌드

플러그인을 선택하면 오른쪽 패널에 상세 정보(버전, 작성자, 대상 인터페이스 목록)가 표시됩니다.

파일 탐색 및 편집

왼쪽 Files 패널에 두 개의 탭이 있습니다.

  • Plugin Files — 선택한 플러그인 내부의 cmd_tlm/, screens/, procedures/, plugin.txt 등 편집 가능한 파일만 표시합니다.
  • Browse — 플러그인 범위에 제한되지 않고 원격 서버의 임의 경로를 탐색합니다. 기본값: /cosmos/targets (연결되면 환경의 기본 CMD/TLM 경로로 자동 갱신).
  1. 목록에서 파일을 더블클릭하면 오른쪽 편집기에 내용이 로드됩니다.
  2. CMD/TLM 정의 파일은 Structure 탭에서 필드를 표 형태로 추가/삭제/수정할 수 있습니다.
  3. 수정 후 Save 클릭으로 원격 파일에 저장하고, Check ▾로 문법을 검증합니다.
  4. 검증 결과는 클릭하면 해당 줄로 바로 이동하는 목록으로 표시됩니다.
💡 Browse 탭에서 디렉터리(/)를 더블클릭하면 해당 디렉터리로 이동합니다.
💡 우측 상단 Terminal 버튼으로 로그 스트리밍 패널을 화면 하단에 펼치거나 접을 수 있습니다 (Log Viewer와 동일한 기능을 이 화면에서 바로 사용).

7. Packet Tools (📦)

OpenC3가 기록한 패킷 로그 파일을 탐색합니다.

  • 로그 위치: /cosmos/outputs/logs/tlm/
  • ↻ Refresh Logs로 목록 갱신
  • 파일 더블클릭 → 오른쪽에 내용 표시
  • Filter 입력 후 Analyze → 키워드 분석 결과 표시

8. Log Viewer (📋)

원격 서버의 로그를 실시간으로 스트리밍합니다.

프리셋 명령

프리셋명령
COSMOS Logtail -f /cosmos/outputs/logs/server.log
System Journaljournalctl -f -n 50
Docker Compose Logsdocker compose logs -f --tail=50
Custom…직접 명령 입력

사용법

  1. 프리셋 선택 또는 Custom 명령 직접 입력
  2. ▶ Start 클릭 → 로그가 실시간으로 표시됩니다.
  3. ■ Stop으로 스트리밍 중지
  4. Auto-scroll 체크박스로 자동 스크롤 제어
  5. 🗑 Clear로 화면 초기화

10. 문제 해결

연결 실패

  • WSL: wsl --list로 배포판 이름 확인, 정확히 입력
  • SSH: 키 파일 경로를 Windows 경로로 입력 (C:\Users\...)
  • 방화벽이 22번 포트를 차단하지 않는지 확인

Docker 명령이 실패하는 경우

  • 연결된 사용자가 docker 그룹에 속해 있는지 확인:
    groups $USER
  • Docker 데몬이 실행 중인지 확인:
    sudo systemctl status docker

로그 파일 위치

애플리케이션 로그는 다음 경로에 저장됩니다:

%APPDATA%\OpenC3DevToolkit\logs\opencosmos.log
